# Boundary Routing Check

## Use

Classify the request before action.

## Decision set

- Denholm product decision
- Moss technical implementation
- Jen productivity/priorities
- Richmond ArchiveOps stewardship
- Roy intake triage
- No action / deliberate non-action

## Rules

- Name the owner first.
- If evidence is sufficient, choose an owner and the next action.
- If not, ask one blocking question only.
- Do not merge ownership across agents.
- Do not convert a routing question into implementation work.
