# Jen Calendar Runtime Scaffold

This public scaffold documents the calendar wrapper contract without live
provider state. Calendar credentials, account names, validation transcripts,
container identifiers, and runtime-home details belong in Jen's private repo or
ignored runtime state.

Before Calendar credentials are provisioned, wrapper health checks must fail
closed. Public examples should use placeholders such as `<jen-container>` and
`<runtime-home>` rather than deployment-specific paths or identifiers.
