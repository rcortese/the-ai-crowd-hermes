# Canonical Message Patterns

Use these patterns when turning classified Todoist date evidence into user-facing language.

## Neutral wording

Use "items with past dates" / "itens com data passada" when Todoist semantics are unclear.
Use "late", "overdue", or "atrasado" only for established `hard_deadline` items.
Avoid generic past-date framing like "atrasados", "overdue pile", "zerar atrasados", or "você acumulou pendências" when the item is not an established hard deadline.

## Category action summary

- `hard_deadline`
- `recurring_hard_obligation`
- `recurring_maintenance`
- `soft_surface`
- `ambiguous`

hard deadline → do/renegotiate/reschedule/set or verify deadline; recurring hard obligation → preserve recurring due and verify/update current-cycle deadline from policy; recurring maintenance → choose next viable occurrence/re-anchor; soft surface → keep today/move to `Esta Semana`/remove due date; ambiguous → ask or suggest a short review.
